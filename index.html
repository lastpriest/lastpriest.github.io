Hexo 文档

发表于 2016-01-04

Hexo 是个轻量级博客框架，基于javascript编写，支持markdown，支持数据迁移，使用简单。

安装

npm install -g hexo-cli

开始

Hexo 安装好后，使用下面的命令初始化bolg环境，建立blog目录：

初始化成功后可以看到下面的目录结构：

文件结构说明

scaffold

Scaffold 是模板目录，从hexo new [layout] <title>中创建的新post，会从该目录中搜索相应的layout，并基于模板创建。

source

Source 目录是放置blog站点内容的地方，默认情况下包含_posts和_drafts两个文件。Hexo会忽略其他带有_前缀的文件。这里的markdown文件和html文件将会被hexo generate渲染处理，渲染后的文件将放置在public目录下。

themes

themes文件夹下放置的是站点的主题文件，可以在github中搜索hexo-theme了解相关。

配置设置

_config.yml文件是站点的全局设定文件。下面列出设置项目的相关字段：

Site

Setting Description
title The title of your website
subtitle The subtitle of your website
description The description of your website
author Your name
language The language of your website. Use a 2-lettter ISO-639-1 code. Default is en.
timezone The timezone of your website. Hexo uses the setting on your computer by default. Some examples are America/New_York, Japan, and UTC.

URL

Setting Description Default
url The URL of your website –
root The root directory of your website –
permalink The permalink format of articles :year/:month/:day/:title/
permalink_default Default values of each segment in permalink –

Website in subdirectory
If your website is in a subdirectory (such as http://example.org/blog) set url to http://example.org/blog and set root to /blog/.

Directory

Setting Description Default
source_dir Source folder. Where your content is stored source
public_dir Public folder. Where the static site will be generated public
tag_dir Tag directory tags
archive_dir Archive directory archives
category_dir Category directory categories
code_dir Include code directory downloads/code
i18n_dir i18n directory :lang
skip_render Paths not to be rendered. You can use glob expressions for path matching –

Writing

Setting Description Default
new_post_name The filename format for new posts :title.md
default_layout Default layout post
titlecase Transform titles into title case? false
external_link Open external links in new tab? true
filename_case Transform filenames to 1 lower case; 2 upper case 0
render_drafts Display drafts? false
post_asset_folder Enable the Asset Folder? false
relative_link Make links relative to the root folder? false
future Display future posts? true
highlight Code block settings –

Category & Tag

Setting Description Default
default_category Default category uncategorized
category_map Category slugs –
tag_map Tag slugs –

Date / Time format

Hexo uses Moment.js to process dates.

Setting Description Default
date_format Date format MMM D YYYY

Pagination

Setting Description Default

Setting Description

hexo命令

初始化

$ hexo init [folder]

初始化并创建站点目录.

创建

$ hexo new [layout] <title>

创建新的post，layout是目录scaffolds目录下的文件，有 page，draft，post。默认是 post。

生成

$ hexo generate

生成静态文件，用于部署发表到服务器。

Option Description
-d, –deploy Deploy after generation finishes
-w, –watch Watch file changes

发表

$ hexo publish [layout] <filename>

发表draft目录下的文件，实际上是将改目录下的文件移动到post目录

本地服务器

$ hexo server

默认为 http://localhost:4000/.

Option Description
-p, –port Override default port
-s, –static Only serve static files
-l, –log Enable logger. Override logger format.

部署

$ hexo deploy

Option Description
-g, –generate Generate before deployment

渲染

$ hexo render <file1> [file2] ...

Option Description
-o, –output Output destination

数据迁移

$ hexo migrate <type>

从其他bolg系统中迁移数据到hexo，如ghost等。需要安装相应的工具。

清理

$ hexo clean

清理缓存 (db.json) 和静态文件 (public).

list

$ hexo list <type>

列出指定的文件，如hexo list post将会列出所以_posts目录下的文件。

选项

安全模式

$ hexo --safe

禁用所有安装的插件和脚本。可以尝试使用该命令定位安装新插件后出现的bug。

Debug 模式

$ hexo --debug

该模式下会生成log文件，终端中会打印log信息。

静音模式

$ hexo --silent

该模式下终端不会显示任何信息。

自定义设置文件位置

$ hexo --config custom.yml

显示草稿

$ hexo --draft

显示 source/_drafts 下的草稿文件.

自定义 CWD

$ hexo --cwd /path/to/cwd
